# 02 - Turn Confirmed Intent Into Accountable Work

**What to build:** A human sponsors one agent, confirms a compact Work Intent, opens a Build Session, and sees the resulting Accountable Work Thread in Butler while identity, intent, and authority remain separate.

**Blocked by:** 01 - Boot One Inspectable Kernel Environment.

**Status:** completed

- [x] Human and agent Principals can be created and inspected.
- [x] Agent Passport binds sponsor, runtime, model/configuration, validity, and permitted intent classes without granting authority.
- [x] Conversation or input creates only a proposed Work Intent.
- [x] Explicit confirmation creates an immutable attributed Work Intent and idempotent receipt.
- [x] Provisional intent permits public discovery but customer context and effect operations remain blocked.
- [x] A Build Session binds exact Principal, Passport, Work Intent, base references, and expiry without storing draft contents.
- [x] Butler creates one Accountable Work Thread showing identity, intent, Build Session, and separate authority status.
- [x] Invalid, expired, or mismatched Passport/Intent requests return structured rejection.
