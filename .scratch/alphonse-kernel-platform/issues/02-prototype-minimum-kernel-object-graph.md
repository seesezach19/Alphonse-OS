Type: prototype

## Question

What is the smallest canonical object graph and set of lifecycle boundaries Kernel must own to enforce the accepted platform invariants while leaving domain meaning and intelligence in user space?

Prototype asset: [Minimum Kernel Object Graph](../prototypes/minimum-kernel-object-graph.md)

## Answer

Kernel owns the minimum deterministic graph needed to identify exact installed definitions, attribute participants and intent, govern context and action authority, admit execution, and preserve operational truth. The graph separates immutable package definitions and deployments; Principal, Passport, delegation, intent, access, approval, activation, and admission authority; and first-class runs, effects, evidence, obligations, escalation, recovery, and ledger transitions.

Build drafts, domain meaning, context payloads, secrets, model reasoning, substrate telemetry, and UI projections remain user space or external-plane state. They cross Kernel boundaries only through typed references, hashes, claims, and validated transition requests.
