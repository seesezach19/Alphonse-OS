# Define The Accountability And Execution Lifecycle

Type: prototype
Status: resolved
Blocked by: 02

## Question

What deterministic states and transitions connect Agent Passports, Work Intents, delegation, capabilities, Accountability Contracts, execution admission, runs, evidence, escalation, corrective work, and recovery?

Prototype asset: [Accountability And Execution Lifecycle](../prototypes/accountability-and-execution-lifecycle.md)

## Answer

The lifecycle preserves an exact independently validated chain from Principal, Agent Passport, Work Intent, Delegation, active Capability, context and credential authority, through one Execution Envelope and one Run, to immediately gated Effects, Evidence Records, Operational Obligations, Escalations, corrective work, and Recovery Cases.

Run execution and Accountability Projection remain separate in every interface. Effects are individually or atomically batch-enumerated before dispatch; uncertainty may be reconciled through append-only evidence without erasing history. Waiver and accepted loss require exact human authority. Corrective work always creates new intent and normal execution authority, while Passport containment remains separate from ordinary failure.
