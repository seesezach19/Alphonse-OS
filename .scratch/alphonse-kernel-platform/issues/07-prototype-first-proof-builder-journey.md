## Question

What complete Builder and agent journey proves that a plain-language workflow can become an active, accountable Operational Package in under one day without modifying Kernel internals?

Prototype asset: [First-Proof Builder Journey](../prototypes/first-proof-builder-journey.md)

## Answer

The qualifying proof uses inventory discrepancy correction. An unfamiliar workflow-aware Builder and a versioned domain-neutral Builder Toolkit must progress from plain-language intent through discovery, context, package authoring, validation, evaluation, publication, staged deployment, exact approval/activation, one-button runtime handoff, staging failure/recovery, and one low-risk reversible production effect.

The workflow-specific path must complete in under eight active hours with under two hours total human attention, zero Kernel/schema changes, no direct authority edits, no secret copying, no hidden provider memory, complete evidence, and preserved failure history. Zach may run an engineering rehearsal, but qualification requires a Builder unfamiliar with Kernel internals.
