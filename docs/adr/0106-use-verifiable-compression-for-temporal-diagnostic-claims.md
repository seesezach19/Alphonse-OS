# Use Verifiable Compression For Temporal Diagnostic Claims

Deterministic software remains the authority for exact evidence, admissibility, state, policy, approval, and effects,
but not every useful diagnostic conclusion is available from a closed mechanical rule. Alphonse may therefore use
agents to interpret frozen evidence or investigate changing external material, provided that intelligence is
compressed into a bounded, attributed, immutable, and independently inspectable claim record.

Processing uses four profiles: D0 closed deterministic computation; D1 deterministic computation over an exact
retrieved snapshot; A1 bounded agent interpretation over frozen evidence; and A2 governed open-world investigation.
`NOT_ESTABLISHED` is a claim-support outcome rather than a fifth processing profile. One investigation may emit
deterministic snapshot facts, agent hypotheses, contradicted claims, and unresolved claims without flattening them to
one confidence score.

Every claim separates production method, verification performed, asserted support, software-evaluated effective
support, evidence status, temporal scope, limitations, supersession, and authority-decision reference. Event or valid
time, observation or retrieval time, system acceptance time, assessment time, and freshness or expiry remain distinct.
Historical evidence is immutable; changed interpretation appears as a new revision that supersedes rather than edits
the prior claim.

Agent sign-off establishes accountable interpretation, not truth or authority. Agents may propose hypotheses,
investigations, and remediations, but software assigns effective assurance and permitted consequence from evidence
admissibility, completeness, freshness, source independence, action risk, capability scope, and approvals. Retrieval
uses governed connectors that preserve exact source identity, bytes or digest, retrieval time, and freshness. Neither
agent confidence nor successful process compliance grants execution authority.

Ticket 09 introduces only the minimal D0 Claim Envelope needed for authenticated observations, interpreted effects,
evaluations, and unresolved conclusions. Ticket 16 strengthens the A1 diagnosis contract with cited claim IDs,
counterevidence, alternatives, falsifiers, explicit non-established items, and a structured next observation. D1 and
A2 retrieval are added later through separately authorized retrieval services rather than general worker network
access.
