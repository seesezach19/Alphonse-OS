# Accept the V0.2 product boundary

V0.2 is complete when a fresh documented installation uses customer-owned n8n and Alphonse Node to authenticate and attribute one successful-but-wrong inventory execution, reproduce its missing-SKU defect, commission a customer-controlled repair worker, independently reject the original and verify an inactive candidate, require human promotion, confirm the target revision, retain rollback and regression evidence, and repeat idempotently. OTLP, Agent SDK, Console, Butler, ALPHONSE_DATA, managed hosting, and broad evaluation are not required for the headless architecture proof.
