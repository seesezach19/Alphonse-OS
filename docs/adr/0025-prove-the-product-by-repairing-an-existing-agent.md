# Prove the product by repairing an existing agent

The decisive product proof is an unfamiliar builder integrating an existing unreliable agent, after which Alphonse detects a silent behavioral failure, opens a causally supported Diagnostic Case, reproduces it, creates a Repair Candidate in the external source repository, passes targeted repair verification, and verifies the deployed revision. Operational Package construction remains important but is no longer the initial proof of value; consequential effects may adopt Kernel admission after diagnostic value is established.
