# Separate Governed Interpretation From Observed Incident Facts

Diagnostic Evidence Packages explicitly separate governed interpretation dependencies, authenticated observations,
deterministic derived facts, and coverage and limitations. Test Orchestrator authorship of preregistered contracts
is governed configuration authorship, not evidence authorship. The package records contract ownership, registration,
activation, observation reporters, deterministic projector and evaluator identities, and packager identity.

For the duplicate-delivery proof, the Behavior Contract declares only an exact workflow and destination scope,
logical-operation correlation, committed CRM-create effect selection, and a count threshold of at most one. The
Integration Behavior Contract declares only exact-string idempotency semantics, repeated-key behavior, designated
append-only ledger identity, commit-record meaning, consistency, and reconciliation behavior. Neither contract may
state retries, observed key scope or count, incident identities, implementation mapping, root cause, or diagnosis.

Restrictive schemas with `additionalProperties: false` prohibit notes, arbitrary metadata, defect paths, root causes,
expected diagnoses, and free-form implementation descriptions. IDs, labels, schema descriptions, comments, fixture
names, policy labels, and filenames remain neutral. Acceptance structurally verifies that the orchestrator authored
only configuration, observations were authenticated under expected observer-specific grants and keys, deterministic
services authored projections and evaluations, the packager introduced no facts, stimulus and verifier produced no
diagnostic material, and worker-visible inputs contain no answer-key fields. HMAC attribution is not exclusive
authorship. Contracts interpret; observations reveal; the worker infers.
