# Build The Vertical Proof In Three Attributable Stages

The first failing black-box acceptance test ends at creation of one immutable unclaimed Diagnostic Assignment and
contacts no model. The orchestrator deploys exact packages, contracts, grants, fixtures, and readiness bindings,
records the Sealed Deployment Manifest, relinquishes credentials, and exits. Separate Scenario Stimulus sends two
bounded webhook requests and exits. Observer and first-party Diagnostic Plane services alone produce receipts,
correlation, committed-effect interpretation, violated evaluation, trigger, complete package, retention pins, and
assignment. The Acceptance Verifier uses only public read-only APIs.

The verifier checks pre-execution workflow binding and runtime matches; exact observer, grant, and schema provenance;
two deliveries and request-key equality relationships under one operation; two ledger-backed committed effects;
adequate coverage; reproducible projection; violated evaluation; expected first-party authorship; answer-free
contracts; bounded disclosure; available pinned artifacts; completion cutoff; immutable assignment bindings; zero
controller-authored diagnostic material; and absence of Worker Runs, broker grants, and model requests.

Test 2 claims that exact assignment and runs one isolated worker. Test 3 uses an explicit Diagnostic Consistency Test
Policy to create three fresh assignments over the same package and applies the preregistered 3/3 rubric. The Test 1
package never changes. Failures therefore attribute cleanly to deterministic diagnostic construction, worker
dispatch and isolation, or model consistency and run reproducibility. The failing Test 1 harness is the first
implementation artifact and fixed target for the vertical slice.
