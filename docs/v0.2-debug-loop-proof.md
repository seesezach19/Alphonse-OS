# V0.2 Repeatable Debug Loop Proof

## Run

Prerequisites: Node.js 22+, Docker Desktop with Linux containers, and Docker Compose.

From a clean checkout:

```powershell
npm install
npm run rehearse:v0.2
```

No database setup is required. The proof creates isolated PostgreSQL, Alphonse Node, Diagnostic Plane, n8n adapter, and customer-owned n8n state, then removes it after each journey.

The rehearsal runs the complete public Debug Loop twice from clean state. Each repetition runs:

1. The normal Owner-authorized repair journey through exact target confirmation.
2. The uncertainty journey covering applied, not-applied, mismatch, reconciliation, and rollback.

It compares normalized public case projections, candidate and verification outcomes, Promotion histories, deterministic revision and rollback artifact digests, and target behavior. UUIDs and timestamps are intentionally excluded.

## Inspect

During a retained local run, the final case is explainable without database or private log access:

```powershell
$env:ALPHONSE_URL="http://127.0.0.1:43206"
$env:ALPHONSE_TOKEN="<bootstrap-owner-token>"
npm run diagnostic:cli -- get-case <case-id>
npm run diagnostic:cli -- get-promotion <promotion-id>
```

`get-case` shows the failure specification, reproduction attempts and bundles, Repair Tasks, candidates, Verification Receipts, Promotions, current projection, and legal next operations. `get-promotion` adds exact Owner authorization, application request, uncertainty, reconciliation, confirmation, rollback reference, receipts, and append-only event history.

## Boundaries

The proof uses deterministic local fixtures only. It performs no AWS activity, real email delivery, production inventory write, private hosted call, or live model inference. Credentials stay in customer-owned adapter configuration.
